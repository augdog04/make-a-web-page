Important Concepts

Lesson 1: The Basics of the Web and HTML

How the Web Works

The web uses a system of computers to communicate with each other. If you go to www.yahoo.com the computer will send a HTTP Request to a server.

The server retrieves the info and sends it back allowing the web browser to interpret and display the page. This video helped me understand the process better.

HTML means Hypertext Markup Language. HTML docs form the bulk of the content of the web.

HTML contains text which is "what you see" and markup or "how it looks."

Tags and Elements

HTML docs consist of HTML elements. When writing HTML, tags and elements are needed. This video goes into greater depth about tags and elements.

Computers are not really stupid but in a way they are as they take instructions literally. One small mistake by a programmer can cause huge problems for a program.

Inline and Block Elements

HTML elements are either inline or block. Block elements create a non-visible box around the content inside of them.

Lesson 2: Creating a Structure Document with HTML

Developer Tools (in Browsers)

HTML elements are either inlin or block. Block elements form boxes around content--the boxes don't actually show up on the screen.

"Tree-like Structure" of HTML

The "tree-like structure" analogy refers to the fact that other elements exist inside of HTML elements...think of a family tree for computers.

In a browser, this displays as boxes inside of boxes...and boxes and boxes and okay you get it.

The relationship between indented HTML and boxes

When reading HTML docs as text, on the left side of the document indentations go up and down. The more indented an element is, the more nested its corresponding "box" is.

Text Editors--For Programming

When writing code, programmers use special text editors (Sublim Text is an example) These editors make the programmer's life easier. For example, some text editors will automatically generate a closing HTML tag when you write an opening tag.

Lesson 3: Adding CSS Style to HTML Structure

Avoiding Repetition

Avoiding repetition is important for a variety of reasons.

Avoiding Errors: When programmers don't avoid repetition, they will often have to do the same thing over and over again. Copying and pasting could be a programmers best friend, or rewriting code with only slight differences. If the programmer changes something, errors although minor could disrupt the feng shui of the project. He or she must go back to make that same change elsewhere.

Consistency: I this web page, I wanted every lesson to look the same. Instead of manually using the same style, I was able to assign each lesson divthe same class and then specify the style for that class in only one place! By avoiding repetion, it's now very easy for me make changes to this style.

CSS

CSS stands for Cascading Style Sheets and they give programmers a way to control the style of related HTML elements. This is done by giving similar HTML elements the same classname and then specifying the style that sould apply to that class.
